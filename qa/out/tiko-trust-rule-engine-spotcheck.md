## Oracle calibration (so the numbers measure resolution, not definition)

The javac oracle was extended during this exercise to count **every type-name position jcma's per-node
scan sees**, because each is a genuine type dependency that a naive oracle under-counts ‚Äî making jcma
look falsely imprecise. Added: sealed **`permits`** entries, **class literals** (`Foo.class`, e.g.
MapStruct `@SubclassMapping(source = FooJpa.class)`), **qualified-name qualifiers** (`Outer` in
`Outer.Inner`), **switch type/record patterns** (`case Foo(int a) ->`), and **method-reference type
qualifiers** (`Type::method`). Before these, jcma's apparent intra precision was ~88%; every "extra"
removed by them was jcma being **correct** and the oracle incomplete. After calibration, intra
precision is 100% and the residual external extras (1.0%) are classpath/edge-case differences.
